Event loop

一、相关概念

1、线程和进程

  • 进程：操作系统分配内存的基本单位，进程所属的内存中保存所有的执行代码和数据；
  • 线程：CPU执行代码的基本单位；位于进程内部，一个进程可能同时存在多个线程，允许单个任务分成不同的部分运行。

2、JavaScript是单线程

JavaScript的主要用途是与用户互动，以及操作DOM。这决定了它只能是单线程。那么，加载耗时的JS任务会长时间占用主线程，导致页面内容产时间没有反应。怎么办？

3、同步和异步

所有任务可以分成两种，一种是同步任务（synchronous），另一种是异步任务（asynchronous）;

总结异步过程的构成要素：发起一个异步请求、工作线程执行异步任务、通知主线程、主线程调用回调函数

二、Event loop 核心

执行脚本时：

1、无异步任务

2、有异步任务

事件循环机制: 就是在执行上下文的过程中，对函数的入栈和出栈。执行前函数先入栈，执行完后函数出栈。如若遇到了一些异步操作像回调函数以及ajax、setTimeout等，会先将他们交给浏览器的其他模块去执行，执行完后，会把回调函数放入到 消息队列 中。当所有的 stack执行完后再开始执行 消息队列 中的函数。

在异步任务完成后，通知主线程，进入消息队列中，那么浏览器是否可以选择先执行哪个任务？

3、Microtask 和 Macrotask

总结

macro-task包括： script(整体代码), setTimeout, setInterval, setImmediate, I/O, UI rendering。

micro-task包括： process.nextTick, Promises, Object.observe, MutationObserver

执行顺序： 函数调用栈清空只剩全局执行上下文，然后开始执行所有的micro-task。当所有可执行的micro-task执行完毕之后。循环再次执行macro-task中的一个任务队列，执行完之后再执行所有的micro-task，就这样一直循环。

三、js异步编程的几种方法

假定有两个函数f1和f2，f1是一个很耗时的任务，f2等待f1的执行结果

1、回调函数

回调函数的优点:简单、容易理解和部署；
缺点：不利于代码的阅读和维护，各个部分之间高度耦合（Coupling），流程混乱。

2、事件监听

优点：易理解，可以绑定多个事件，每个事件可以指定多个回调函数，而且可以”去耦合”（Decoupling），有利于实现模块化。
缺点：整个程序都要变成事件驱动型，运行流程会变得很不清晰。

3、发布/订阅

4、Promises对象

优点:回调函数变成了链式写法，程序的流程可以看得很清楚

5、web worker

  • Web Worker无法访问DOM节点,BOM树；
  • Web Worker无法访问全局变量或是全局函数；

兼容性：IE10及以上；

四、由Event loop机制导致的js定时动画问题

原因：无论是 setTimeout() 还是 setInterval() ，基于 js 的 Event loop 的运行机制，实际上指是把动画代码添加到浏览器UI线程队列中以等待执行；
解决：html5相关API——requestAnmiationFrame();
